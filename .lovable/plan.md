

# תוכנית תיקון מקיפה — ניווט מובייל + באגים נוספים

## בעיות שנמצאו

### בעיה 1 (קריטית) — z-index חוסם לחיצות בתפריט מובייל

**זו הבעיה העיקרית.** התפריט ההמבורגר נמצא בתוך ה-`<nav>` ב-`z-50`, אבל:
- כפתור הצ'אט: `fixed bottom-6 right-6 z-[9998]`
- בועת הפרואקטיבית: `fixed bottom-24 right-6 z-[9997] w-[300px]`
- באנר ה-Cookies: `fixed bottom-0 z-50`

על מסך מובייל (375px), הבועה הפרואקטיבית תופסת כמעט את כל רוחב המסך ויושבת **מעל** התפריט. כפתור הצ'אט גם הוא מעל התפריט. כל אלה גורמים ללחיצות על פריטי הניווט "לא לעבוד" — הלחיצה נתפסת ע"י האלמנט העליון, הניווט לא קורה, והתפריט נשאר פתוח.

**פתרון:** כשהתפריט המובייל פתוח — להסתיר את כפתור הצ'אט, הבועה הפרואקטיבית, ובאנר הקוקיז. שתי גישות אפשריות:

- **גישה א (מועדפת):** להעביר את מצב `mobileOpen` דרך context או prop כדי שהצ'אטבוט וקוקי יוכלו להסתיר את עצמם.
- **גישה ב (פשוטה יותר):** להעלות את ה-z-index של התפריט המובייל ל-`z-[9999]` ולהוסיף backdrop שחוסם את כל מה שמתחת.

הגישה הפשוטה ביותר: להרים את ה-z-index של ה-mobile menu container ל-`z-[9999]` ולהוסיף overlay/backdrop מתחתיו שמכסה את כל המסך.

### בעיה 2 — באג ב-regex של linkify (קישור WhatsApp מהבוט)

בקובץ `ChatMessage.tsx`, הפונקציה `linkifyContent` משתמשת ב-`URL_REGEX.test(part)` עם regex גלובלי (`/g`). הבעיה: `test()` על regex עם `/g` מזיז את `lastIndex` — מה שגורם לזיהוי URL לעבוד ולא לעבוד לסירוגין.

**פתרון:** כש-`split()` מקבל regex עם capturing group, התוצאה היא מערך שבו אינדקסים אי-זוגיים הם תמיד URL-ים. להחליף את ה-`test()` בבדיקת אינדקס:
```
parts.map((part, i) => i % 2 === 1 ? <a ...>{part}</a> : part)
```

### בעיה 4 (קטנה) — CSS conflict: `block` ו-`flex` על אותו אלמנט

בשורה 186 ב-Navbar, הקישורים הרגילים בתפריט מובייל כוללים גם `block` וגם `flex`. זה לא שובר דברים, אבל צריך לנקות.
